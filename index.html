Ferramentas de docking

Qual ferramenta devo usar?

Escolha a ferramenta adequada para seu estudo de docking molecular baseado no tipo de ligante e requisitos específicos do seu projeto.

AutoDock Vina

Ideal para ligantes pequenos e médios. Rápido, preciso e amplamente utilizado. Melhor escolha para estudos gerais de docking molecular.

Características:

  • Não lida bem com flexibilidade do receptor
  • Não suporta docking de peptídeos
  • Parâmetros de força fixos
  • Dependência da Caixa de Encaixe
  • Flexibilidade Limitada do Receptor

AutoDock 4

O AutoDock 4 é a versão clássica que utiliza algoritmo genético Lamarckiano (LGA) para busca conformacional. É mais lento que o Vina, mas oferece maior controle sobre os parâmetros.

Características:

  • Controle detalhado dos parâmetros
  • Mapas de energia pré-calculados
  • Bem estabelecido na literatura
  • Mais lento que o Vina
  • Configuração mais complexa
  • Requer geração de grid maps

AutoDock CrankPep

Especializado em docking de peptídeos. Considera a flexibilidade da cadeia principal, essencial para modelagem de peptídeos.

Características:

  • Considera flexibilidade de backbone
  • Algoritmo específico para peptídeos
  • Baseado no AutoDock 4
  • Maior custo computacional
  • Ideal quando flexibilidade de backbone é crítica
  • Docking com peptídeos de 20 aminoácidos de comprimento
💡 Recomendação

Para a maioria dos projetos, comece com AutoDock Vina. É a ferramenta mais moderna, rápida e com melhor custo-benefício entre precisão e tempo computacional.

Terceira etapa: Grid box

📦 O que é Grid Box?

Grid Box é uma caixa tridimensional (3D) definida sobre a proteína que delimita o espaço de busca do docking.

1 Selecione os resíduos que compõem o sítio

Na barra lateral à esquerda (guia Dashboard) clique no mais (+) ao lado do receptor para exibir os resíduos que constituem seu receptor.

Posteriormente selecione os resíduos que constituem seu sítio de ligação, clicando no círculo à direita (select/deselect (S)).

Vídeo: Seleção dos resíduos do sítio ativo
🔢 Resíduos utilizados

106, 109, 110, 111, 162, 163, 164, 247, 248, 264, 268, 271, 272, 286

2 Determinar tamanho da caixa e o centro da caixa

Determine o tamanho da caixa e o centro da caixa com base nos resíduos selecionados e guarde as informações em um arquivo conf.txt

Grid → Grid Box
💡 Dica

Em View → Show box as lines facilita a exibição da caixa

Vídeo: Definindo o Grid Box

No arquivo conf.txt deve conter exatamente o nome do ligante, receptor, os parâmetros do Grid Box e exhaustiveness.

receptor = 6WX4.pdbqt
ligand = geranyl_isobutanoate.pdbqt

out = res_out.pdbqt

center_x = 11.938
center_y = -25.901
center_z = -37.072

size_x = 56
size_y = 72
size_z = 70

exhaustiveness = 8
⚙️ Exhaustiveness

Quanto maior o exhaustiveness, mais tempo o Vina gasta explorando diferentes posições, orientações e conformações do ligante no Grid Box.

Instalação do AutoDock CrankPep

💻 Requisitos

Linux ou WSL (Windows Subsystem for Linux)

1 Baixe e conceda permissões

Baixe o ADFRsuite_Linux-x86_64_1.0_install e execute:

chmod a+x ADFRsuite_Linux-x86_64_1.0_install
./ADFRsuite_Linux-x86_64_1.0_install
2 Instalação

Navegue ao diretório de instalação escolhido e execute:

bash install.sh
⚠️ Variáveis de ambiente

Ao fim da instalação, será mostrado como atualizar a variável de ambiente PATH no arquivo .cshrc ou .bashrc

3 Configurar PATH
nano ~/.bashrc

Adicione ao final do arquivo:

export PATH=/home/seuusuário/ADFRsuite-1.0/ADFRsuite_x86_64Linux_1.0/bin:$PATH

Salve e atualize:

source ~/.bashrc
4 Teste da instalação
which adcp

Deve retornar:

/home/seuusuário/ADFRsuite-1.0/ADFRsuite_x86_64Linux_1.0/bin/adcp

Redocking com AutoDock CrankPep

1 Preparação da pasta de trabalho

No seu computador crie uma pasta de trabalho (ex: ADCP_red)

2 Preparo do PDB

No PyMol:

  • Limpe a estrutura PDB (água e ligantes indesejados)
  • Separe a proteína do peptídeo
  • Adicione ambos os arquivos .pdb na pasta ADCP_red
3 Preparo do Redocking

No terminal, navegue até ADCP_red e execute:

# Protonar receptor e peptídeo
reduce 3I5R.pdb > 3I5R_H.pdb
reduce peptide.pdb > peptide_H.pdb

# Converter para .pdbqt
prepare_receptor -r 3I5R_H.pdb
prepare_ligand -l peptide_H.pdb

# Gerar target file (.trg)
agfr -r 3I5R_H.pdbqt -l peptide_H.pdbqt -o 3I5R
📦 Target file (.trg)

Contém: estrutura do receptor, mapas de afinidade (AutoGrid4), tipos atômicos suportados, pontos de translação (AutoSite) e metadados do docking.

4 Executar o docking
adcp -t 3I5R.trg -s krplpplps -N 20 -n 1000000 -o 3I5R_redocking -ref peptide_H.pdb
📄 Saída

Retorna:

  • Estruturas finais ranqueadas após clustering (3I5R_redocking_*.pdb)
  • Trajetórias Monte Carlo (3I5R_redocking_*.out)
  • Arquivos principais (3Q47_redocking_ranked_*.pdb)
  • Tabela de resultados no terminal com pose, afinidade (kcal/mol) e RMSD

Docking AutoDock CrankPep

1 Preparação da pasta de trabalho

No seu computador crie uma pasta de trabalho (ex: ADCP_dock)

2 Preparo do PDB

No PyMol:

  • Limpe a estrutura PDB (água e ligantes indesejados)
  • Remova o peptídeo da proteína
  • Adicione o arquivo .pdb na pasta ADCP_dock
3 Preparo do docking

No terminal, navegue até ADCP_dock e execute:

# Protonar receptor
reduce 3I5R.pdb > 3I5R_H.pdb

# Converter para .pdbqt
prepare_receptor -r 3I5R_H.pdb
Vídeo: Preparo do receptor para docking
4 Gerar target file (.trg) manualmente

No terminal execute:

agfrgui
1. Em Receptor pdbqt adicione seu receptor protonado
2. Na engrenagem em ligand binding pockets selecione version 1.1
3. Clique em Compute pockets [AutoSite 1.1] e selecione seu sítio
4. Em docking box clique em "box all displayed ligand binding pocket" e ajuste o tamanho com padding
5. Clique em generate target file ... → renomeie (ex: 111.trg) → Save
Vídeo: Criação manual do target file
5 Executar o docking
adcp -t 111.trg -s aqtqslvypfpgpipk -N 20 -n 50000 -o pep1_docking > pep1.log
Argumento Descrição
-t arquivo target
-s sequência do peptídeo
-N número de réplicas (buscas independentes)
-n passos de Monte Carlo por réplica
-o prefixo dos arquivos de saída
-ref estrutura de referência (para RMSD)

Redocking no AutoDock Vina ou AutoDock 4

🎯 Objetivo

Não é prever a conformação do ligante, mas sim avaliar se o algoritmo consegue reproduzir uma pose conhecida no mesmo sítio.

1 Preparo do receptor

Realize a preparação do receptor conforme os tutoriais do AutoDock Vina ou AutoDock 4. Salve em .pdbqt em sua pasta de trabalho.

2 Criação do Grid

Crie o grid ao redor do ligante cristalográfico.

3 Arquivo de configuração
receptor = 6WX4.pdbqt
ligand = redocking.pdbqt

out = res_out.pdbqt

center_x = 10.233
center_y = -26.861
center_z = -37.2

size_x = 40
size_y = 40
size_z = 40

exhaustiveness = 8
4 Preparar ligante cristalográfico
1. No PyMol extraia o ligante cristalográfico do PDB
2. No AutoDockTools:
Ligand → Input → Open → ligante.pdb → Open
Ligand → Torsion Tree → Choose Torsions → Make all active bonds non-rotatable → Done
Ligand → Output → Save as Pdbq → redocking.pdbqt
📁 Localização

Salve na pasta validacao

Vídeo: Preparo do ligante cristalográfico
5 Execução do Docking

Utilize o mesmo processo de execução descrito nos tutoriais do AutoDock Vina ou AutoDock 4.

6 Avaliação do RMSD

No PyMol:

File → Open → 6WX4.pdb # Complexo cristalográfico
File → Open → res_out_ligand_1.pdbqt # Pose 1 do resultado docking

# Na janela de comando PyMol, execute:
align 6WX4, res_out_ligand_1
Vídeo: Avaliação do RMSD
RMSD (Å) Interpretação
≤ 1.0 Å Excelente
≤ 2.0 Å Aceitável / Validado
2.0–3.0 Å Questionável
> 3.0 Å Falha no protocolo
