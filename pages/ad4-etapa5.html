Quinta etapa: Preparação dos arquivos com Raccoon | AutoDock VC

1 Navegue até o diretório de instalação do mgltools
Linux → cd /usr/local/mgltools_i86Linux2_1.5.4/bin/
Windows → cd C:\Program Files (x86)\MGLTools-1.5.7
2 Execute Raccoon

Adicione o script raccoon.py no diretório do mgltools

pythonsh raccoon.py # Linux
python.exe raccoon.py # Windows
3 Navegue pelas guias da interface
3.1 Ligand(s) → Add ligands ou Add directory → selecione ligantes ou pasta com ligantes → Abrir
3.2 Receptor(s) → Add receptor file → selecione o receptor → Abrir
3.3 Maps → Affinity maps → Load template → selecione grid.gpf → Abrir
3.4 Docking → select docking setup → From template → Load template → selecione dpf.dpf
⚠️ Arquivo necessário

Caso seja requisitado o AD4.1_bound.dat clique em OK → selecione AD4.1_bound.dat → Abrir

3.5 VS Generation → Set directory → selecione a pasta resultados → GENERATE
💡 Importante

A pasta Resultados deve estar vazia

4 Executar o docking em todos os ligantes

Abra o terminal e navegue ao diretório resultados:

Linux:
cd ~/Dock4/resultados
bash RunVS.sh

Windows:
cd %USERPROFILE%\Dock4\resultados
RunVS.bat
